SpringMVC框架学习与企业级课程知识复习

发表于 2019-09-02 | 更新于 2020-08-03

由于之前面试被问及SpringMVC当中的8大注解问题时直接懵逼，开始了解新式SpringMVC框架，同时温习《企业级应用软件开发》课程所学的知识

SpringMVC设计概念

最初的样子

在早期的java Web开发中，将所有的操作交由JavaBean来处理，这种模式称为$Model1$

弊端：
1.JSP 和 Java Bean 之间严重耦合，Java 代码和 HTML 代码也耦合在了一起
2.前端和后端相互依赖，前端需要等待后端完成，后端也依赖前端完成，才能进行有效的测试
3.代码难以复用

改进后的版本

将模型和操纵器分离开来，形成早期的MVC模型，称之为Model2

目前个人所参与的项目全部都是采用这种架构，所以并不知晓注解的用法

最新的SpringMVC架构

为解决持久层中一直未处理好的数据库事务的编程，又为了迎合 NoSQL 的强势崛起，Spring MVC 给出了方案

特点：
1.结构松散，几乎可以在 Spring MVC 中使用各类视图
2.松耦合，使得各个模块分离
3.与Spring无缝集成

Spring Boot

什么是Spring Boot

Spring Boot 是由 Pivotal 团队提供的全新框架，其设计目的是用来简化新 Spring 应用的初始搭建以及开发过程。该框架使用了特定的方式来进行配置，从而使开发人员不再需要定义样板化的配置。用我的话来理解，就是 Spring Boot 其实不是什么新的框架，它默认配置了很多框架的使用方式，就像 Maven 整合了所有的 Jar 包，Spring Boot 整合了所有的框架。
