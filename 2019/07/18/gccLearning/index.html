关于DHCP

DHCP的中文名称是$\color{red}{动态主机配置协议}$，全称是Dynamic Host Configuration Protocol
DHCP的主要作用是集中的管理、分配IP地址，使网络环境中的主机动态的获得IP地址、Gateway地址、DNS服务器地址等信息，并能够提升地址的使用率

DHCP报文详解

如下图所示为DHCP报文的格式

字段 说明
op 和BOOTH兼容，只有BOOTHREQUEST=1和BOOTHREQUEST=2两个取值，具体的消息类别在尾部的options中
htype 硬件类型代码
hlen 硬件地址长度(系统目前只对10mb以太网支持，硬件地址长度固定为6)
hops 客户端清0。DHCP服务器在提供中继服务时使用（记录跳数）
xid 一个客户端软件产生的随机数，用于识别请求和应答消息匹配
secs 客户进入IP申请进程的时间或者更新IP地址进程的时间；由客户端软件根据情况设定
flags 标志字段，这个16比特的字段目前只有最左边一位有用
ciaddr 客户的IP地址，只有客户端处于BOUND,RENEW,REBINDING的状态下发送消息的时候才设置。可以用来响应ARP协议
yiaddr 由DHCP服务器分配给客户端的IP地址
siaddr 表明DHCP协议下一个阶段要用的服务器的地址
giaddr DHCP中继服务器的IP地址
chaddr 客户端的硬件地址
sname 服务器的主机名
file 启动文件名字
options 除IP地址外的所有其他子选项都放在这里

DHCP特点

DHCP的主要功能
1.DHCP允许计算机快速、动态的获得IP地址
2.DHCP报文除了能获取IP地址之外，还能获取例如子网掩码等租用地址的详细信息

DHCP服务的优点
1.网络管理员可以验证IP地址和其他配置参数，而不用去检查每一个主机
2.DHCP不会同时租借相同的IP地址给两台主机
3.DHCP管理员可以约束特定计算机使用特定的IP地址
4.可以为每个DHCP作用域设置很多选项
5.客户端在不同子网之间移动时不需要重新设置IP地址

DHCP服务的缺点
1.DHCP不能发现网络上非DHCP客户机已经在使用的IP地址
2.当网络上存在多个DHCP服务器的时候，一个DHCP服务器不能查出已经被其他服务器租用出去的IP地址
3.DHCP服务器不能跨客户机通信，除非路由器允许DHCP转发
