初识DHCP技术（其实计网学过）

看一看菊花关于DHCP技术的白皮书菊花牛逼

关于DHCP

DHCP的中文名称是$\color{red}{动态主机配置协议}$，全称是Dynamic Host Configuration Protocol
DHCP的主要作用是集中的管理、分配IP地址，使网络环境中的主机动态的获得IP地址、Gateway地址、DNS服务器地址等信息，并能够提升地址的使用率

DHCP报文详解

如下图所示为DHCP报文的格式

字段 说明
op 和BOOTH兼容，只有BOOTHREQUEST=1和BOOTHREQUEST=2两个取值，具体的消息类别在尾部的options中
htype 硬件类型代码
hlen 硬件地址长度(系统目前只对10mb以太网支持，硬件地址长度固定为6)
hops 客户端清0。DHCP服务器在提供中继服务时使用（记录跳数）
xid 一个客户端软件产生的随机数，用于识别请求和应答消息匹配
secs 客户进入IP申请进程的时间或者更新IP地址进程的时间；由客户端软件根据情况设定
flags 标志字段，这个16比特的字段目前只有最左边一位有用
ciaddr 客户的IP地址，只有客户端处于BOUND,RENEW,REBINDING的状态下发送消息的时候才设置。可以用来响应ARP协议
yiaddr 由DHCP服务器分配给客户端的IP地址
siaddr 表明DHCP协议下一个阶段要用的服务器的地址
giaddr DHCP中继服务器的IP地址
chaddr 客户端的硬件地址
sname 服务器的主机名
file 启动文件名字
options 除IP地址外的所有其他子选项都放在这里

DHCP特点

DHCP的主要功能
1.DHCP允许计算机快速、动态的获得IP地址
2.DHCP报文除了能获取IP地址之外，还能获取例如子网掩码等租用地址的详细信息

DHCP服务的优点
1.网络管理员可以验证IP地址和其他配置参数，而不用去检查每一个主机
2.DHCP不会同时租借相同的IP地址给两台主机
3.DHCP管理员可以约束特定计算机使用特定的IP地址
4.可以为每个DHCP作用域设置很多选项
5.客户端在不同子网之间移动时不需要重新设置IP地址

DHCP服务的缺点
1.DHCP不能发现网络上非DHCP客户机已经在使用的IP地址
2.当网络上存在多个DHCP服务器的时候，一个DHCP服务器不能查出已经被其他服务器租用出去的IP地址
3.DHCP服务器不能跨客户机通信，除非路由器允许DHCP转发

DHCP Client

DHCP Client是DHCP服务中的客户端，也是整个DHCP服务中活动的触发者和驱动者，DHCP Client的活动过程为：
1.DHCP Client开始申请IP地址时，发出$\color{red}{广播}$的DISCOVER报文，收到回应的OFFER，然后发出$\color{red}{广播}$的REQUEST报文，收到回应的ACK，这样就得到了地址
2.对SERVER分配的IP地址进行有效性检测，若分配的IP地址不可用，则自动迁移到初始状态重新申请IP地址
3.达到续约时间（租约的一半）后，发出$\color{red}{单播}$的REQUEST报文进行续约，如收到ACK后更新租约，如收到NAK后，则重新发起申请过程
4.如果单播续约报文没有响应，达到重新绑定时间后（租约的$\frac{7}{8}$），会发送$\color{red}{广播}$的续约报文
5.Client重启后不进行DISCOVER申请，而是直接发送REQUEST报文给SERVER
6.Client在进行申请地址时，报文中的request IP address 会填入之前使用过的IP地址，如果这时SERVER认为可用那么就会进行分配，如果该地址有人使用了，SERVER回应NAK，Client收到之后重新进行申请，该字段设置为空

DHCP Server

DHCP Server是DHCP服务的提供者，它通过DHCP报文与DHCP Client沟通，为各种类型的用户分配合适的IP地址，并根据需要进行地址池和其他网络参数的相关配置。DHCP Server的主要功能为

创建和删除地址池

网络管理员在DHCP Server上创建地址池，当Client提出请求的时候，从地址池中选出空闲的IP地址以及其他参数给Client，$\color{red}{一个SERVER可能有一个或多个地址池}$

配置地址池相关参数

系统可以对地址池配置DNS、Domain Name、WINS、netbios-node-type等参数，用于Client的网络配置

处理Client发来的DHCP报文

DHCP Server一共接收以下5种DHCP报文
1.DHCP_DISCOVER
2.DHCP_REQUEST
3.DHCP_DECLINE
4.DHCP_RELEASE
5.DHCP_INFORM
对每一种报文的处理如下

DHCP_DISCOVER

1.检查地址池是否有可分配地址，如果没有，则向系统管理员报告；如果有，按照如下优先顺序进行选择：①.当前绑定中的当前地址；②.Client以前的地址（现在已经expire或是release了，并且available的）③.在Requested IP Address option中并且valid and available的；④.从地址池中分配一个可用地址，根据消息所接收到的子网字段或是根据中继代理的地址进行分配
2.选定租约：根据选定IP地址来决定用户的绑定租约
3.处理Client的其他网络参数
4.发送回应的OFFER报文

DHCP_REQUEST

Request的来源：
1.Client响应Server的DHCP_OFFER
2.Client确认一个先前分配的IP地址
3.Clent请求延长某个网络地址的LEASE
如果DHCP_REQUEST请求包含一个$\color{red}{SERVER\ \ IDENTIFIER\ \ OPTION}$，则是情形1，否则是情形2或3
如果Request中包含有CLIENT_IDENTIFIER或者LIST_OF_REQUESTED_PARAMETERS，那么Cient必须在以后的报文中都包含它们（或者其中的一个）
DHCP_SERVER根据收到的Request报文的类型进行不同的处理，若可以分配合法的IP地址，则回应一个DHCP_ACK报文段

DHCP_DECLINE

若Client发现Server分配的IP地址不可用，则向Server发送一个DHCP_DECLINE报文，Server收到之后将该网络标记为NOT_AVAILABLE

DHCP_RELEASE

Server将Client回应的非法网络地址标记为NOT ALLOCATED，它同时应该保持这个Client的初始化参数，以便将来的REUSE

DHCP_INFORM

Client向Server发送DHCP_INFORM报文用于请求Server分配给Client租约的详细信息，包括所属地址池的详细配置信息等等

给Client分配一个可用的IP地址

Server按照以下优先级为Client分配IP地址
1.按照Server数据库中与Client绑定的地址
2.Client以前曾使用过的地址
3.DHCP_DISCOVER包中请求的地址
4.地址池中可用的地址顺序查找，将最先找到且没有冲突的地址返回
5.检测冲突地址链中的冲突地址，如果可以用，则重新分配给用户
6.从正常分配出去的地址中为Client的DISCOVER请求分配地址
7.如果未找到可用地址，向管理员报错

为Client延长租期请求

Client在分配租约使用一半时间后，会主动向Server申请继续使用该IP地址，Server对Client的续约请求进行处理，若发现请求的IP地址可用，则回应ACK给Client，告知其可以继续使用该IP地址，并更新租约和定时器信息

设定或释放保留地址

保留地址是DHCP协议的IP地址池中不分配的地址段，一旦设定为保留地址后，这个区间的IP地址就不再参加整个IP地址池的分配而保留起来，将保留地址的起始地址和结束地址记录到该IP地址池的参数中

探测分配IP地址的活动情况

在DHCP_SERVER向客户端分配IP地址时，Server首先需要确认所分配的IP没有被网络上的其他设备使用，这就需要Server发送ICMP的数据包来对所分配的IP地址进行探测，如果IP地址可用，Server将继续完成IP地址的配置。否则报告出错信息

配置相关网络参数以告知Client

1.配置Client网关信息
2.配置Client使用的DNS服务器
3.配置Client网络使用的域名
4.配置Client的NETBIOS服务器IP地址
5.配置Client所对应的NETBIOS节点类型
