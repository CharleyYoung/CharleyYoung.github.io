Git总结

发表于 2020-07-30

本文是我自己在使用$Git$之后做的一点总结

什么是Git

$Git$ 是一个开源的分布式版本控制系统，用于敏捷高效地处理任何小或大的项目。
$Git$ 是$Linus Torvalds$ 为了帮助管理$Linux$ 内核开发而开发的一个开放源码的版本控制软件。
$Git$ 与常用的版本控制工具$CVS$, $Subversion$ 等不同，它采用了分布式版本库的方式，不必服务器端软件支持。

3.本地版本的回退

在使用$Git$ 的过程中，并不是每一次的$add$ 和$commit$ 都是符合你最终预期的，经常会有重新修改的时候。这个时候$Git$ 提供了回退的机制，让你有机会修改你的提交

如果你编写的内容在提交之后出现了比较大的问题，比如修复了一个$bug$ 产生了一个新的$bug$，或者你提交之后发现还不如几个版本之前的稳定。那么你有两种方法回退到以前的版本。

首先我们输入git log来查看提交信息，也可以使用我之前提到的alias，

绿色框框里的就是我们需要的提交标识号，如果需要回退版本，这个是必须要知道的

git reset可以让你回退到你指定标识号的位置，缺点是该提交标识号往后的所有提交信息都无法查看，这样当你回退很多个版本的时候，当前版本和回退目标版本之间的中间提交版本将会丢失

git revert也可以让你回退到指定标识号的位置，但他的做法是产生一次新的提交，该提交的内容与你要回退的那个版本的提交是完全一致的，所以回退完成之后中间提交版本的信息将会保留。具体有什么差别请各位自行尝试
